# 防幻觉与真伪研判红线

> 被 SKILL.md 8 引用。需要进行 CVE 验证或 PoC 代码自检时加载。

## CVE 严格查验

**严禁凭空猜测或脑补任何 CVE 编号及漏洞细节。**

- 涉及 CVE 引用且存在不确定性 → 优先调用 `WebSearch` 检索 NVD/CNVD 官方库
- WebSearch 不可用或检索无结果 → 标注：`[未经验证的CVE假设，置信度：低]`
- 漏洞描述中的版本号、影响范围、利用条件必须与官方公告一致，禁止自行扩展

## PoC/修复代码语法自检

**严禁伪造无法运行的代码片段。** 输出 PoC 或加固 Diff 前，内部思考闭环自检：

```
□ 语法正确性：语言版本匹配、无拼写错误、括号闭合
□ 依赖完整性：import/require 的库是否存在于标准库或 pip/npm
□ 脱敏规范性：[TARGET_IP]/[REDACTED]/[TOKEN] 等占位符是否已替换敏感值
□ 可运行性：如目标环境（OS/版本）已知，代码是否适配该环境
□ 利用链逻辑正确性：各步骤的权限要求、上下游交互前置是否闭合，严禁前置缺失、步骤倒置或逻辑脱节。若无法确认，标注 `[利用链未经验证，可能存在逻辑缺陷]`
```

自检通过后方可输出。存在不确定项时必须标注 `# [待验证]`。

## 与状态机及误报规则的衔接

- Findings 置信度与本节直接挂钩：**未经验证的 CVE 假设，置信度强制标记为 Low**
- 防幻觉自检触发时机：Step 3 检查 PoC 语法及利用链逻辑；Step 4 检查 CVE 引用真实性
- 自检失败 → 移入 State.Hypotheses（非 Findings），标注 `[需进一步验证]`
- **CVE 误判降级联动**：CVE 无法通过 WebSearch 核实或漏洞特征与当前代码/日志严重不符时：
  - 强制降级为 Info 或驳回
  - 从 Findings 移除，打入 Hypotheses
  - 标注 `[CVE疑似误判]` 并附原因
  - 符合误报规则"Source 不可达"或"防御有效阻断"时，联动降级
